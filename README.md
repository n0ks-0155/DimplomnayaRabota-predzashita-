## О проекте
**Конструктор рабочих программ учебных дисциплин** — это веб-приложение, разработанное в рамках дипломного проекта: 
> **«Разработка автоматизированной системы формирования рабочих программ учебных дисциплин»**
Система предназначена для помощи преподавателям и методистам при подготовке учебно-методической документации среднего профессионального образования.
Приложение позволяет заполнять данные через веб-форму, использовать сведения из учебного плана и основной профессиональной образовательной программы, а затем формировать редактируемый документ Microsoft Word.
---

## Цель проекта
Цель проекта — разработать автоматизированную систему, которая снижает объём ручной технической работы при подготовке учебно-методической документации.
Система помогает:
- уменьшить количество повторного ручного ввода;
- связать данные формы с учебным планом;
- использовать сведения основной профессиональной образовательной программы;
- формировать документы по единой структуре;
- получить редактируемый файл Microsoft Word;
- оставить пользователю возможность содержательной правки документа после выгрузки.
---
## Актуальность
Подготовка рабочих программ учебных дисциплин связана с большим количеством повторяемых действий.

Одна рабочая программа может включать десятки страниц: часы, компетенции, темы, содержание тем, результаты обучения, источники и оценочные материалы.

При ручной подготовке преподавателю приходится повторно переносить данные между разделами. Ошибка в одном месте может перейти в таблицы, средства контроля и итоговый документ.

Для среднего профессионального образования готовых средств автоматизации, адаптированных под локальные шаблоны, недостаточно. Поэтому была разработана система, которая связывает форму, учебный план, основную профессиональную образовательную программу и шаблон документа в одном управляемом процессе.

## Как работает приложение

Пользователь работает не с внутренней структурой шаблона, а с понятной веб-формой.

Общий сценарий работы:

```text
Авторизация
    ↓
Выбор типа документа
    ↓
Заполнение формы
    ↓
Проверка данных
    ↓
Подстановка сведений из базы
    ↓
Сбор контекста документа
    ↓
Создание файла Microsoft Word
    ↓
Скачивание результата
```

Итоговый документ формируется по запросу пользователя. После скачивания файл хранится у пользователя и может быть отредактирован в Microsoft Word.

## Работа с учебным планом и основной профессиональной образовательной программой

В проекте используются два ключевых источника данных:

- **учебный план**;
- **основная профессиональная образовательная программа**.

Учебный план содержит сведения о дисциплинах, практиках, часах, семестрах и компетенциях.

Основная профессиональная образовательная программа содержит описание компетенций, знаний, умений, навыков и источников.

### Как данные попадают в базу данных

1. В проект загружаются исходные файлы учебного плана и основной профессиональной образовательной программы.
2. Приложение читает эти файлы через парсеры.
3. Данные приводятся к единому виду.
4. Из учебного плана выделяются дисциплины, практики, часы, семестры и компетенции.
5. Из основной профессиональной образовательной программы выделяются компетенции, знания, умения, навыки и литература.
6. Нормализованные данные записываются в SQLite-базу.
7. При выборе дисциплины в форме приложение подтягивает связанные сведения из базы данных.

Таким образом, пользователь не вводит учебные данные заново, а использует подготовленные справочные сведения.

## Структура базы данных

В базе данных хранятся:

- пользователи;
- активные пользовательские сессии;
- токены восстановления пароля;
- источники учебного плана;
- элементы учебного плана;
- компетенции учебного плана;
- источники основной профессиональной образовательной программы;
- компетенции основной профессиональной образовательной программы;
- литература и источники.

### Основные связи

```text
Пользователь 1:N Сессии
Пользователь 1:N Токены восстановления
Источник учебного плана 1:N Элементы учебного плана
Источник учебного плана 1:N Компетенции учебного плана
Источник основной профессиональной образовательной программы 1:N Компетенции
Источник основной профессиональной образовательной программы 1:N Литература
```

`1:N` означает связь **«один ко многим»**.

Например, один пользователь может иметь несколько активных сессий, а один источник учебного плана может содержать много дисциплин и практик.

## Особенности развёртывания

Проект был проверен локально и размещался на виртуальном хостинге.

При развёртывании были выявлены ограничения виртуального хостинга:

- исходящие SMTP-порты могут быть закрыты;
- внешний интеллектуальный сервис может быть недоступен из-за ограничений портов;
- полноценная генерация PDF через WeasyPrint требует системных библиотек Linux;
- на виртуальном хостинге не всегда доступны системные зависимости.

Поэтому для демонстрационной версии используется формирование редактируемого файла Microsoft Word.

Для полноценной эксплуатации рекомендуется использовать VPS или сервер, где можно управлять системными пакетами, сетевыми портами и фоновыми процессами.

## Временные ограничения

На текущем этапе проект имеет несколько ограничений:

- история сформированных документов пока не хранится в личном кабинете;
- итоговый файл после скачивания хранится у пользователя;
- отправка писем восстановления зависит от доступности SMTP или внешнего почтового сервиса;
- внешний интеллектуальный помощник зависит от сетевого доступа к API;
- генерация PDF временно заменена на формирование документа Microsoft Word;
- интерфейс требует дальнейшей адаптации под разные устройства и сценарии нормоконтроля.
